# Product Identity

## Core Position

Time Tracker is not primarily a product that provides an AI service itself.

Its core job is to help a person produce the highest-quality personal data that their own AI can understand, trust, reuse, and act on well.

## Decision Principle

When making product decisions, prefer the option that improves data quality over the option that only looks smart on the surface.

That means prioritizing:

- clear structure over flashy AI-like behavior
- reliable capture over premature automation
- exportability and reuse over app-local lock-in
- user-owned context over provider-owned inference
- consistency over short-term novelty

## Anti-Shallow Checklist

Before accepting a feature, UI change, or workflow shortcut, ask:

1. Does this help the user generate better data for their own AI?
2. Does this preserve meaning, context, timestamps, and intent in a reusable form?
3. Does this reduce ambiguity, or does it only make the product feel smarter?
4. If the app's built-in AI disappeared, would this still be a valuable capability?
5. Are we helping the user think, record, and hand off better, instead of replacing that with shallow convenience?

If the answer is weak, the decision is probably drifting away from the product's identity.

## Reminder For Future Work

Do not optimize for "an app that does AI for the user" unless that directly strengthens the real goal.

Optimize for "an app that leaves the user with clean, rich, durable data that any personal AI can use exceptionally well."
